# 小程序商城 - 产品需求与范围

## 一、项目背景
这是一个小程序商城项目，当前目标是先完成第一版 MVP，优先打通小程序端与后端接口闭环。

用户在小程序端登录后，可以完成以下操作：
- 商品浏览
- 商品详情查看
- 加入购物车
- 提交订单
- 发起支付
- 查看订单
- 取消订单
- 地址管理

当前阶段优先实现后端接口与前台小程序页面联调，后台管理端后续再逐步补齐。

## 二、第一版功能范围
第一版包含以下功能：

1. 微信登录
2. 商品浏览
3. 商品详情
4. 购物车
5. 提交订单
6. 微信支付
7. 订单查询
8. 订单取消
9. 地址管理
10. 支付超时关单
11. 支付成功异步处理

## 三、第一版不包含
以下内容第一版明确不做：

1. 后台管理页面
2. 退款
3. 退货 / 售后
4. 优惠券
5. 秒杀
6. 分销
7. 多商户

## 四、小程序页面范围
第一版页面包括：

1. 首页 / 商品列表页
2. 商品详情页
3. 购物车页
4. 确认订单页
5. 支付结果页
6. 我的订单页
7. 订单详情页
8. 地址管理页

## 五、当前开发目标
当前项目目标不是一次性做完整电商平台，而是：

- 先完成最小可用闭环
- 先跑通商品 -> 购物车 -> 下单 -> 支付 -> 订单查询
- 后续再补后台管理与高级功能

## 六、开发优先级
建议优先顺序：

1. 公共层与基础设施
2. 用户与地址
3. 商品模块
4. 购物车模块
5. 订单模块
6. 支付模块
7. MQ 模块
8. 小程序联调
